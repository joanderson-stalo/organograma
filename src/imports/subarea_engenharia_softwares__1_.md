# Subárea de Engenharia de Softwares - Confrapag

## Visão Geral
A subárea de **Engenharia de Softwares** da Confrapag é responsável por **criar novos produtos**, **implementar conexões com novos fornecedores** e **desenvolver soluções tecnológicas** que impulsionam o crescimento da empresa.

Ela atua como o núcleo de **inovação e desenvolvimento**, sendo responsável pela criação de novas soluções, testes de tecnologias emergentes e definição de novas arquiteturas.

---

## Estrutura Organizacional

### Squads
- Equipes multidisciplinares formadas por:
  - Engenheiros de Software
  - UX/UI
  - QA
  - Product Owner (PO)
- Responsáveis por desenvolver produtos ou soluções específicas

### Squad LAB (Laboratório de Aplicações)
- Equipe dedicada à:
  - Criação de aplicações experimentais
  - Desenvolvimento de novas integrações
  - Testes de novas tecnologias

---

## Estrutura de Cargos

### 1. Engenheiro de Software

#### Front-end
Responsável pela criação das interfaces dos sistemas e aplicativos.

- Transforma designs do UX/UI em interfaces funcionais
- Foco em:
  - Experiência do usuário (UX)
  - Interface do usuário (UI)
- Tecnologias:
  - React
  - React Native
  - Next.js

#### Principais responsabilidades:
- Desenvolvimento de interfaces interativas
- Componentização para reutilização de código
- Otimização de performance
- Consumo de APIs internas (Back-end)

---

#### Back-end
Responsável pela lógica, dados e infraestrutura das aplicações.

- Atua como o “motor” do sistema
- Tecnologias:
  - PHP
  - Laravel
  - Banco de dados SQL

#### Principais responsabilidades:
- Implementação de regras de negócio
- Integração com APIs internas e externas
- Comunicação com banco de dados
- Testes (Unit e Feature)
- Segurança (autenticação, autorização)
- Otimização de performance

---

#### Mobile
Responsável pelo desenvolvimento de aplicações móveis e embarcadas.

- Atua em:
  - Smartphones
  - Tablets
  - POS (maquinetas)

#### Tecnologias:
- Android: Kotlin / Java
- iOS: Swift / Objective-C
- Híbrido: React Native

#### Principais responsabilidades:
- Desenvolvimento de apps nativos e híbridos
- Integração com APIs Back-end
- Garantia de performance e segurança
- Comunicação com hardware (NFC, chip, tarja)
- Publicação e versionamento de aplicativos

---

### 2. UX/UI Designer

Responsável pela experiência e interface do usuário.

#### UX (User Experience)
- Define fluxo de navegação
- Foco na usabilidade e lógica do sistema

#### UI (User Interface)
- Define aparência visual
- Elementos de interação

#### Principais responsabilidades:
- Criação de protótipos
- Definição de fluxos de navegação
- Garantia de usabilidade
- Uso do Figma para design e prototipagem

---

### 3. Arquiteto de Software (Software Architect)

Responsável pela visão técnica de alto nível dos sistemas.

#### Principais responsabilidades:
- Definir arquitetura dos sistemas
- Garantir:
  - Escalabilidade
  - Segurança
  - Performance
  - Custo

#### Atuação estratégica:
- Criar diagramas e especificações
- Orientar desenvolvedores
- Validar decisões técnicas com o CTO
- Participar do roadmap técnico
- Identificar gargalos e propor melhorias

---

### 4. Quality Assurance (QA)

Responsável pela garantia da qualidade do software.

#### Principais responsabilidades:
- Prevenção de defeitos durante o desenvolvimento
- Validação das regras de negócio

#### Tipos de testes:
- Testes manuais
- Testes automatizados (Selenium, Cypress, APIs)
- Testes de regressão

#### Atuação:
- Trabalha junto com Produto e Engenharia
- Melhora contínua do processo de desenvolvimento

---

### 5. Especialista de Implementação Técnica (EI)

Responsável pela implantação técnica das soluções da Confrapag.

#### Objetivo:
Garantir implantação eficiente, segura e padronizada das soluções.

#### Principais responsabilidades:
- Onboarding técnico de clientes e fornecedores
- Integração de APIs
- Acompanhamento de integrações
- Diagnóstico de falhas técnicas
- Criação de playbooks e documentação
- Automação de processos de implantação
- Comunicação entre áreas técnicas e de negócio

---

### 6. Gestor de Gateways e Processamento

Líder técnico e estratégico dos sistemas de pagamento.

#### Principais responsabilidades:
- Supervisão de sistemas críticos:
  - Pagamentos
  - Captura
  - Processamento de transações

#### Gestão de equipes:
- Lidera Squads multifuncionais:
  - Product Owner
  - Engenheiros de Software
  - UX/UI

#### Atuação estratégica:
- Alinhar tecnologia com objetivos de negócio
- Melhorar:
  - Uptime
  - Performance
  - Custos de transação
- Garantir estabilidade e inovação dos sistemas

---

## Relações com Outras Áreas

- **Produto (PO / PM)**
  - Definição de features e prioridades

- **Suporte Técnico**
  - Recebe feedbacks de problemas em produção

- **Sustentação**
  - Recebe bugs para correção

- **CTO**
  - Validação técnica e direcionamento estratégico

---

## Objetivo da Área

Desenvolver soluções tecnológicas escaláveis, seguras e inovadoras, garantindo a evolução contínua dos produtos da Confrapag e a integração eficiente com parceiros e fornecedores.
